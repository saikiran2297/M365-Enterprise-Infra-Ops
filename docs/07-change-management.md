# Change Management

## Change types
- Standard: pre-approved, low risk, repeatable
- Normal: requires CAB/approval
- Emergency: SEV-driven, with retrospective approval and PIR

## Required change artefacts
- Risk assessment
- Implementation steps + validation
- Rollback plan
- Comms plan
- Monitoring impact statement

## Repo workflow
- Create PR for each change
- Use PR template (risk/rollback/testing)
- Tag reviewers (simulated)
- Link change issue/template
